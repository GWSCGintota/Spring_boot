# 34. Database Migration

Do not depend on Hibernate automatic schema updates for controlled production deployments.

Use a migration tool such as:

- Flyway
- Liquibase

## 34.1 Flyway dependency

```xml
<dependency>
    <groupId>org.flywaydb</groupId>
    <artifactId>flyway-core</artifactId>
</dependency>
```

Migration files:

```text
src/main/resources/db/migration/
├── V1__create_products_table.sql
├── V2__add_product_description.sql
└── V3__create_category_table.sql
```

Example:

```sql
CREATE TABLE products (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(120) NOT NULL,
    price DECIMAL(12, 2) NOT NULL,
    quantity INTEGER NOT NULL
);
```

Recommended production configuration:

```yaml
spring:
  jpa:
    hibernate:
      ddl-auto: validate
```

Advantages:

- Version-controlled schema
- Repeatable deployments
- Reviewable database changes
- Reliable environment consistency
- Easier rollback planning
