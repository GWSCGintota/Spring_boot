# 2. Spring Framework vs Spring Boot

## 2.1 Spring Framework

The Spring Framework provides the fundamental programming model used by Spring applications.

Important Spring Framework features include:

- Inversion of Control
- Dependency Injection
- Aspect-Oriented Programming
- Transaction management
- Spring MVC
- Data access
- Validation
- Security integration
- Testing support

Traditional Spring applications often required considerable manual configuration.

## 2.2 Spring Boot

Spring Boot simplifies Spring application development by adding:

- Auto-configuration
- Starter dependencies
- Embedded web servers
- Opinionated defaults
- Externalized configuration
- Production monitoring
- Executable JAR packaging
- Development tools

## 2.3 Comparison

| Feature | Spring Framework | Spring Boot |
|---|---|---|
| Configuration | More manual configuration | Auto-configuration |
| Dependency setup | Individual dependencies | Starter dependencies |
| Web server | Usually configured separately | Embedded server |
| Packaging | Commonly WAR | Commonly executable JAR |
| Production monitoring | Added manually | Actuator support |
| Development speed | Moderate | Faster |
| XML configuration | Supported | Usually unnecessary |
| Convention | Flexible | Opinionated defaults |

Spring Boot does not replace Spring Framework. It provides a simpler and faster way to use it.
